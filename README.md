# LLM-driven-Verification-Assistant

This is a Multi modal LLM agent to assist with RTL Verification.
VLSI Agentic Verification Environment - Setup Guide


1. Multi-Agent Frameworks

pyautogen: Microsoft's AutoGen framework. This is the highly recommended library for your project because of its strong support for conversational loops, iterative debugging, and native execution of Bash/Python scripts (which you need for running VCS/ModelSim).

crewai: An alternative multi-agent framework that uses a strict role-based hierarchy. It is included here in case you want to experiment with it, though AutoGen is generally better suited for the dynamic "compile-fail-fix" loop of VLSI.

2. LLM APIs

anthropic: The official Python SDK for interacting with Anthropic's Claude models. You will use this to connect your agents to Claude 3.5 Sonnet (the recommended model for RTL and Testbench generation) and Claude 4.5/3 Opus (for the Supervisor agent).

python-dotenv: A standard helper library to securely load your API keys (like ANTHROPIC_API_KEY) from a .env file so you don't accidentally hardcode them into your scripts or upload them to GitHub.

3. Waveform Parsing

vcdvcd: A lightweight Python library for parsing Value Change Dump (.vcd) waveform files generated by simulators.

pyvcd: An alternative, robust library for reading and writing .vcd files.
You can use either vcdvcd or pyvcd to build the required Python utility that converts binary waveform data into a text-based truth table so your Simulation Agent can read the exact port values at specific timestamps (e.g., 100ns to 120ns).

Installation Instructions
Run the following command in your terminal to install all dependencies:

# Activate the conda environment
conda activate llm_verification_agent 

# Install dependencies distributed on PyPI
pip3 install -r requirements.txt

# After running the above commands, both pip and conda available for use depending on the package availability with that particular package manager
